Documentation Navigator - User Guide

The Documentation Navigator is a Streamlit app that provides an interactive browser for all folders and files in the workspace's Documentation directory. It lets you quickly find and read documentation without leaving the Snowsight environment.

Prerequisites

  • Access to a Snowflake Workspace that contains this project.
  • The Documentation folder must exist as a sibling directory at the workspace root (alongside the doc-navigator app folder).
  • A running compute pool (SYSTEM_COMPUTE_POOL_CPU) and query warehouse (COMPUTE_WH) assigned to your account.

How to Launch the App

  1. Open your Snowflake Workspace in Snowsight.
  2. Navigate to the doc-navigator folder in the file explorer.
  3. Open streamlit_app.py.
  4. Click the Run button in the top-right corner of the editor.
  5. The app will start and display in the preview pane.
Note: If this is the first time running the app, the compute pool may take a moment to resume. Subsequent launches will be faster.

Using the App

The interface is divided into two panels:

Left Panel - Navigation

  • Folders appear as expandable sections (click to expand/collapse).
  • Each file is shown as a button with an icon indicating its type.
  • Click any file button to load its contents in the right panel.

Right Panel - Content Viewer

When you select a file, its contents are rendered based on the file type:

File Type Icon How It Renders
Markdown (.md) 📝 Formatted rich text with headers, lists, and links
HTML (.html) 🌐 Rendered HTML in a scrollable embedded frame
Notebook (.ipynb) 📓 Raw JSON source displayed in a code block
Python (.py) 🐍 Syntax-highlighted Python code
SQL (.sql) 🗃️ Syntax-highlighted SQL code
Other 📄 Plain text in a code block

Adding New Documentation

The app dynamically reads the Documentation folder each time it loads. To add new content:

  1. Place new files or folders inside the Documentation directory at the workspace root.
  2. Supported formats: .md, .html, .ipynb, .py, .sql, or any text-based file.
  3. The app will automatically pick them up on the next interaction (no code changes needed).
Important: The navigator reads one level of subfolders. If you create nested subfolders (e.g., Documentation/Agents/Advanced/), only files directly inside Agents/ will appear. Move files up or create a new top-level folder instead.

Troubleshooting

Issue Solution
"No Documentation folder found" Ensure the Documentation folder exists at the workspace root as a sibling to doc-navigator.
App fails to start Verify the compute pool is active and your role has USAGE on both the warehouse and compute pool.
File content does not display Check that the file is text-based and UTF-8 encoded. Binary files are not supported.
Changes not appearing Click Run again after modifying snowflake.yml or pyproject.toml. For documentation file changes, simply interact with the app (click a different file and back).
